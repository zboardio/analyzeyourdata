# Як використовувати Analyze Your Data

## Швидкий огляд

**Analyze Your Data** дозволяє завантажувати дані, досліджувати їх в інтерактивній таблиці та створювати до 3 незалежних графіків — все у вашому браузері. Жодні дані не зберігаються на сервері; все залишається у вашій сесії.

---

## Крок 1: Завантажте свої дані

Виберіть одне з підтримуваних джерел даних:

### Пряме завантаження файлу
- Клацніть область завантаження або перетягніть файл
- **Підтримувані формати:** Excel (`.xlsx`, `.xls`), CSV (`.csv`, `.txt`, `.log`), JSON, Parquet, HDF5, SQLite (`.db`, `.sqlite`, `.sqlite3`)
- Для файлів CSV/TXT/LOG підтвердіть або змініть роздільник (кома, крапка з комою, табуляція, вертикальна риска або пробіл)
- Максимальний розмір файлу: **{{VALUE_MAX_FILE_SIZE_MB}} MB**

### База даних SQLite
- Завантажте файл `.db`, `.sqlite` або `.sqlite3`
- Перегляньте доступні таблиці з кількістю рядків і стовпців
- Виберіть таблицю, яку хочете проаналізувати, і натисніть **Load Selected Table**

### Microsoft SharePoint / OneDrive — Припинено

**Microsoft вимкнув неавтентифікований доступ до API спільного доступу OneDrive.** Кінцева точка API, яка раніше дозволяла завантажувати файли за публічними посиланнями SharePoint/OneDrive, тепер повертає помилки автентифікації. Це зміна, здійснена Microsoft — не цим додатком.

Замінне рішення від Microsoft вимагає автентифікації Azure AD OAuth 2.0, яка додає значні незручності (вхід з обліковим записом Microsoft, схвалення адміністратором організації) з обмеженими гарантіями довгострокової стабільності.

> **Рекомендована альтернатива:** Завантажте файл зі SharePoint/OneDrive на комп'ютер, а потім скористайтеся **Прямим завантаженням файлу** вище. Це швидше, надійніше, і ваші дані залишаються повністю під вашим контролем.

### Airtable

Підключення Airtable вимагає **Personal Access Token** і **Base ID**.

#### Як створити Personal Access Token

1. Перейдіть на [airtable.com/create/tokens]({{URL_DOCUMENTATION_AIRTABLE_TOKENS}}) (або перейдіть до Обліковий запис → Developer Hub → Personal Access Tokens)
2. Натисніть **Create new token**
3. Дайте йому назву (наприклад, "Analyze Your Data")
4. У розділі **Scopes** додайте як мінімум:
   - `data.records:read` — для читання записів таблиці
   - `schema.bases:read` — для перегляду таблиць у базі
5. У розділі **Access** виберіть конкретні бази, до яких ви хочете підключитися
6. Натисніть **Create token** і негайно скопіюйте його — ви не зможете побачити його знову

> **Довідка:** [Creating Personal Access Tokens — Airtable Support]({{URL_DOCUMENTATION_AIRTABLE_PAT}})

#### Як знайти ваш Base ID

1. Відкрийте вашу базу Airtable у браузері
2. Подивіться на URL: `https://airtable.com/appXXXXXXXXXXXXXX/...`
3. Base ID — це частина, що починається з `app` (наприклад, `appXXXXXXXXXXXXXX`)

#### Завантаження даних

1. Введіть ваш **Personal Access Token** у поле токена
2. Введіть ваш **Base ID**
3. Натисніть **Connect to Airtable** — відобразиться список доступних таблиць
4. Виберіть таблицю та натисніть **Load Selected Table**

> **Порада:** Ваш токен зберігається тільки в пам'яті сесії браузера — він ніколи не зберігається на сервері. Закриття вкладки браузера очищає його.


> **Порада:** Для конфіденційних або приватних даних використовуйте Пряме завантаження файлу — ваші дані ніколи не покидають браузер.

## Крок 2: Обробка дати та часу (за бажанням)

Обробка datetime **вимкнена за замовчуванням**. Коли вимкнено, ваші дані завантажуються безпосередньо в таблицю — жодних додаткових кроків не потрібно.

Якщо ваші дані містять стовпець datetime і ви хочете аналіз на основі часу:

1. Перемкніть обробку datetime на **Enabled**
2. Виберіть **Datetime Column** із випадаючого списку
3. Виберіть відповідний **Datetime Format** (або введіть власний формат Python `strftime()`)
4. Натисніть **Load data to AgGrid Table**

Згенеровані стовпці включають: `tsYear`, `tsMonth`, `tsDay`, `tsHour`, `tsMinute`, `tsDayOfWeek`, `tsWeekNumber`, `tsDate` та інші.

## Крок 3: Досліджуйте свої дані в AG Grid

**AG Grid** надає потужні можливості інтерактивного дослідження даних з вбудованими бічними панелями:

- **Сортування** — клацніть будь-який заголовок стовпця
- **Фільтр** — клацніть значок фільтра на будь-якому заголовку стовпця, щоб встановити умови, або використовуйте **Панель фільтрів** праворуч для керування всіма фільтрами стовпців в одному місці
- **Групування** — перетягніть заголовки стовпців у панель "Row Group" над таблицею
- **Зведення** — увімкніть режим зведення з **Панелі стовпців** праворуч для перехресних таблиць
- **Зміна розміру** — перетягуйте межі стовпців для регулювання ширини
- **Агрегація** — під час групування AG Grid показує проміжні та загальні підсумки
- **Панель стовпців** — перемикайте видимість стовпців, змінюйте порядок стовпців та налаштовуйте параметри зведення/значень з бічної панелі
- **Панель фільтрів** — переглядайте та керуйте всіма активними фільтрами по всіх стовпцях з однієї зручної панелі

> **Ключовий момент:** Графіки нижче читають з **поточних відфільтрованих/згрупованих даних**, видимих в AG Grid. Кожна дія фільтрування, сортування або групування миттєво оновлює всі графіки — **це основна потужність інструменту.** Використовуйте AG Grid як інтерактивний слайсер даних і бачте результати, відображені в реальному часі у всіх ваших візуалізаціях.

### Експорт з AG Grid

Використовуйте кнопки **Export to Excel** та **Export to CSV** під AG Grid для завантаження поточних видимих даних:

- Експорт завжди відображає **поточне відображення** AG Grid — фільтри, групування та сортування враховуються
- **Експорт в Excel** включає форматування таблиці з активними фільтрами, тому ви можете продовжити фільтрування безпосередньо в Excel
- **Експорт у CSV** надає чистий плоский файл відфільтрованих даних
- Це означає, що ви можете застосовувати різні критерії фільтрів в AG Grid та експортувати кілька разів для створення **окремих файлів для різних підмножин** ваших даних — потужний робочий процес для аналізу даних та звітності

> **Порада:** Ви також можете клацнути правою кнопкою миші будь-де в таблиці AG Grid для додаткових параметрів експорту через контекстне меню.

## Крок 4: Створюйте графіки

Ви можете створити до **3 незалежних графіків**, кожен зі своєю конфігурацією:

1. **Показати/Приховати** — використовуйте перемикач, щоб показати або приховати кожну секцію графіка
2. **Тип графіка** — виберіть із: Scatter, Scatter (multi y), Line, Bar (grouped), Bar (stacked), Histogram (grouped), Histogram (stacked), Pie, Bubble, Heatmap, Log, Sunburst, Icicle
3. **Стовпець осі X** — виберіть стовпець для горизонтальної осі
4. **Стовпець(ці) осі Y** — виберіть один або кілька стовпців для вертикальної осі
5. **Стовпець кольору** (за бажанням) — розфарбуйте точки даних за категоріальним стовпцем
6. **Стовпець осі Z** (за бажанням) — для типів графіків Bubble і Heatmap
7. **Заголовки** — встановіть власний заголовок графіка, заголовок осі X та заголовок осі Y

Графіки читають з поточних відфільтрованих/згрупованих даних AG Grid. **Кожна дія фільтрування, сортування або групування в AG Grid миттєво оновлює всі графіки.**

## Крок 5: Експорт

### Окремі графіки
- Натисніть **Download Chart as HTML** під кожним графіком, щоб зберегти його як окремий інтерактивний HTML файл

### Усі графіки (ZIP)
- Натисніть **Download All Charts** вгорі або внизу розділу графіків
- Кожен активний графік експортується як окремий самостійний HTML-файл, об'єднані в одне ZIP-завантаження
- До ZIP включаються лише графіки з даними

### Дані AG Grid
- Використовуйте кнопки **Export to Excel** або **Export to CSV** під AG Grid (див. Крок 3 вище)
- Експортує точно ті дані, які наразі видимі в AG Grid (враховує фільтри, групування, сортування)

> **Порада:** Експортовані HTML файли графіків повністю інтерактивні — ви можете збільшувати, наводити курсор для підказок і панорамувати — не потрібно програмне забезпечення, лише веб-браузер.

## Поради та усунення неполадок

| Проблема | Рішення |
|---|---|
| Завантаження файлу не вдається | Перевірте, що файл менше {{VALUE_MAX_FILE_SIZE_MB}} MB і в підтримуваному форматі |
| Посилання SharePoint не працює | Microsoft вимкнув неавтентифікований доступ до API. Завантажте файл і скористайтеся Прямим завантаженням файлу. |
| Google Sheet не завантажується | Переконайтеся, що спільний доступ встановлено на "Будь-хто з посиланням може переглядати" |
| Airtable не підключається | Перевірте, що ваш Personal Access Token має області `data.records:read` і `schema.bases:read`, а Base ID починається з `app` |
| Помилки розбору datetime | Перевірте, що вибраний формат відповідає вашим даним. Спробуйте власний формат, якщо потрібно |
| Графіки порожні | Переконайтеся, що дані завантажені в AG Grid та вибрані стовпці X/Y |
| AG Grid не показує дані після фільтрування | Очистіть або налаштуйте ваші фільтри стовпців у Панелі фільтрів |

## Конфіденційність даних

- Усі завантажені дані обробляються **тільки в пам'яті** (ніколи не записуються на диск або в базу даних)
- Дані зберігаються у вашій **сесії браузера** — закриття вкладки очищає все
- Жодні завантажені дані не надсилаються зовнішнім сервісам
- Зберігаються лише добровільні відгуки та анонімна аналітика використання
